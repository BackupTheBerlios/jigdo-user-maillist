On Mon, Feb 25, 2008 at 11:17:17PM -0500, Eric A. Cottrell wrote:
> Hello,
> 
> In doing some more research I noticed that jigdo-mirror sets the
> --no-check-files in the jigdoFile variable.  The help for jigdo-file says:
>   --no-check-files [make-template,md5sum] when used with --cache,
>                    [make-image] Do not verify checksums of files
> 
> This seems to say disable an operation the program should perform in the
> case of MD5Sum.

Ah yes - you're correct in making that connection! On the other hand, this 
means that once you resolve the tmpDir issue, md5sum will also work. 
Basically, jigdo-file first looks in the cache to see if the md5sum for 
that file is already there, and only reads the file if it isn't. But it 
cannot access the cache file in your case, so it aborts.
