I came across this post
http://bmvbab.blogspot.com/2007_04_01_archive.html which suggests
using aria2 instead of wget for downloads.

I haven't tried it and I don't know how well it works. I do know aria2
is a great downloader and can do multi-source downloads and parallel
downloads.

I also found this request to integrate jigdo & metalink:
http://developer.berlios.de/feature/?func=detailfeature&feature_id=3793&group_id=694

metalinks are an XML list of files used by download apps.

.jigdo & .metalink files both contain a lot of the same info, like
mirrors & checksums. the jigdo file has some information that
metalinks don't, like the image/template file which I believe has the
boot info of an ISO.
metalinks can contain some info that .jigdo doesn't, like chunk
checksums (for download repair, nice for large files).

the .metalink will be more verbose/larger, but can be compressed in
transfer. the .metalink can also be used by a lot of standard download
apps.

> - While the Fedora Project uses "MirrorManager" to direct clients to a
> (list of) local mirrors, we would like Jigdo to be able to 'cache' the
> mirror list returned by MirrorManager and use these mirrors constantly,
> instead of hitting the mirrorlist for each request (e.g. each file). Not
> being able to do so, we fear, results in overloading MirrorManager as
> per ISO image, approximately 4000 requests would have to be made.

MirrorManager has planned support for generating .metalinks for ISO files.
