On Tue, Aug 19, 2008 at 12:38:38AM -0700, Neil M. wrote:
> Interesting idea, I got even more interesting results.  Make-template 
> finds all but 3 of the files from the original .jigdo file.  I have 
> verified that my code writes these files to the image.  Then I'm left 
> with a 13 MB .template file.  Gzip compressed original .template is 
> BIGGER, about 26 MB.  Any idea how that works?

OK, that can happen: These days, Debian images are not generated with 
"make-template" anymore, but with a special hacked version of mkisofs which 
directly outputs .template files. The algorithm used by "make-template" can 
sometimes miss files, especially if they start with lots of zero bytes 
(for Debian, primarily floppy images). "make-template" also has a minimum 
size below which it leaves files in the template. So if just a few small 
files are missing, that's to be expected.

> Not in this case, the decompressed template checks out.  The size matches 
> that described in the .template data header.  Or am I missing something 
> else there?

Nothing that's obvious to me... You'll have to go through your .iso and 
compare it section by section to the original (i.e. use the offsets 
generated by "make-template", or the offsets from the original template 
file.) :-/
