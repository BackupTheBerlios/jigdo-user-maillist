Hi all, I'm trying to implement Jigdo support in my Metalink Checker
code, which has grown into a generic download app.  I've gotten to the
point of downloading/checksumming/decompressing/assembling everything
but it just isn't quite working yet.

So what I've done is download all the files, search for the MD5SUM in
the template and generate a list of byte indexes.  Then what I do is
open a new file (the final ISO), decompressing the template as I go
between checksums.  I replace the checksums with the contents of the
associated file.  The decompressed template (minus data file checksums)
 matches the expected size according to the jigdo file (when present).

I believe my problem can be broken down into this equation (all byte sizes):

decompressed template - file checksums + each downloaded file < original
ISO download

I think I'm missing adding something on the left side of the equation?
Funny thing is whatever I'm generating appears to be a valid ISO file
with all the files in it, but its not the same file as the original ISO.

You can view my assembly logic here, mkiso(), line 748.  Yes, I just
asked you to look at someone else's code, I don't enjoy it either, but
its only 30 lines:

http://metalinks.svn.sourceforge.net/viewvc/metalinks/checker/xmlutils.py?view=markup

You can even run the program if you like, checkout and run console.py
with any .jigdo file.

Thanks in advance, I'm hoping if I get this included in Metalink Checker
it will also feed upstream to GGet and make Jigdo downloads really easy
on everyone.

Neil
