As its name says, the aim of jigdo is to distribute the image in pieces,
so if you want to distribute a non splittable unit of information,
it won't be very effective.
But it does not change the point: you may distribute either your
filesystem.squashfs file as is, or the file, the template and the jigdo files,
but you don't need to publish the .iso file in the latter case.

If I want to publish a tree (say, bar/) into pieces, here are the steps I use  
 - go to a place with enough room to work (e.g. /local/
rsync -a bar /local
cd /local
 - make iso file
mkisofs -v -J -r -d -D -f -l -L -N -o foo.iso bar 
 - make jigdo stuff
jigdo-file make-template -f --image=foo.iso --jigdo=/local/foo.jigdo \ 
--template=/local/foo.template --label Label=/local/ \
 --uri Label=http://<where_to_publish>  /local//bar/
 - publish
copy foo.jigdo foo.template bar/ in http://<where_to_publish>
